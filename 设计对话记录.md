# 文字奇幻剧情冒险游戏：设计对话记录

> 记录从项目导入后的首次讨论，到“界隙书房”大厅原型落地前的设计决策。

## 项目初始状态

用户导入了一个空的 Material 3 Expressive + Jetpack Compose Android 项目。

项目结构是单模块 Gradle Android 应用，入口为 `MainActivity.kt`，使用 Kotlin、Jetpack Compose、Material 3。项目没有 README，也没有 Web 入口或已配置的 Replit Web 工作流。

## 用户：空项目是否适合做文字奇幻剧情冒险游戏？

这个方向很合适。空的 Material 3 Expressive + Jetpack Compose 很适合先建立一套有辨识度的“叙事空间”，而不是被模板化的卡片列表限制住。

## 设计建议：主页面不是菜单，而是故事中的地点

传统文字冒险游戏常见的布局是：

```text
头像
角色名
体力 / 货币
继续游戏
章节列表
背包
设置
```

这种布局把游戏拆成功能模块，却没有让玩家感受到“我正身处这个世界”。

建议将主大厅设计为一个会呼吸、会变化、会记住玩家行为的叙事场所。玩家不是看到一排按钮，而是进入一个临界空间。

## 推荐概念：活着的书房 × 心智剧场

表面上，大厅是一间安静的奇幻书房：

- 桌面
- 营火
- 打开的书
- 漂浮的文字片段

但随着剧情推进，玩家会发现这间书房其实属于角色的内心，是一个“记忆避难所”。

对应关系：

- 书页记录发生过的事件。
- 家具会随着玩家选择变化。
- 被遗忘的人可能出现在房间角落。
- 书架上的空位代表失去的记忆。
- 副本入口可能从书页变成一扇门。
- 当角色精神状态恶化时，书房逐渐变成剧场。

这个概念既保留了文字冒险的阅读属性，又不把大厅做成章节列表。

## 大厅的具体设计

### 中央：叙事核心

中央不是普通头像卡片，而是代表角色精神状态的核心。第一版使用“记忆避难所”作为核心文案：

> 我还记得她的名字

它周围有呼吸式光环、弧线和柔和的琥珀光，代表角色仍在坚持。

### 副本入口：雾中的无名门

副本入口不是“开始剧情”按钮，而是一扇正在形成的门：

- 标题：雾中的无名门
- 副标题：第三幕 · 未完成
- 文案：有人在城墙外叫你的名字

触碰它会触发门的回应，并显示“下一幕正在翻页……”。之后可接入真正的剧情副本。

### 余响

左侧保留过去选择的残影，而不是普通剧情记录：

- 她把戒指放在窗台，没有带走。
- 你没有回头。
- 城墙外的声音，和你一模一样。

### 行囊

行囊是会被带入下一幕的遗物选择：

- 旧银戒
- 空白页
- 灰烬瓶

### 誓言

誓言是会影响后续选择的心理状态：

> 我会找到她，无论门后是什么。

## Material 3 Expressive 的使用方向

不把 Material 3 Expressive 简化为多放圆角卡片，而是使用：

- 富有弹性的空间变化：入口漂浮、光环呼吸、展开层滑入。
- 形状作为叙事语言：安全使用柔和圆形，未知使用不对称边缘，危险使用断裂和尖角。
- 动态颜色系统：以深色墨面为基底，用琥珀表达火光，用雾白承载文字，用紫色表达未知和心智层。
- Material 3 的 Surface、Button、OutlinedButton、Typography、颜色系统承载交互，同时用 Compose Canvas 和动画做出游戏化表达。

## 第一版交互

- 点击“余响”：展开记忆回声面板。
- 点击“行囊”：展开遗物选择面板。
- 点击“誓言”：展开当前誓言面板。
- 点击“雾中的无名门”：触发“门正在回应”的仪式化过渡。
- 点击右上角菜单：显示提示，不让系统菜单抢走故事焦点。
- 所有面板都从底部带有缓动地展开，可收起。

## 本轮实现边界

本轮先做完整可运行的单屏大厅原型，不接入真实存档、剧情数据库、角色系统或副本页面。后续接入真实数据时，应保留大厅的渐进展开和叙事化入口，避免退回传统功能卡片网格。

## 从开始到现在的对话摘要

### 用户

询问这个空的 Material 3 Expressive + Jetpack Compose 项目是否适合开发文字奇幻剧情冒险游戏，并希望设计一个玩家进入游戏后、前往副本前进行准备的大厅页面。用户特别要求跳出常见的“头像、资源、章节列表、背包、设置”布局，提出大胆的设计想法。

### 助手

确认技术方向适合，并提出把大厅做成“界隙营地”：它不是功能菜单，而是一个会呼吸、会变化、会记住玩家行为的叙事场所。随后探索了三种方向：

1. **活着的书房**：副本是打开的书，记忆是书页，装备是桌上遗物。
2. **心智剧场**：大厅是角色内心的舞台，被背叛或失去的人会出现在观众席。
3. **漂浮的世界残骸**：每个副本是一块漂浮的岛屿，选择会改变岛屿之间的连接。

最终推荐将前两者结合：**活着的书房 × 心智剧场**。表面是书房，深层是角色的记忆避难所；书页记录事件，家具和文字会受选择影响，副本入口从书页或裂隙中形成。

### 用户

同意按推荐方向制作，并要求把交互和动画做好，充分利用 Material 3 Expressive；同时要求把从开始到现在的聊天记录以 Markdown 文档保存到项目根目录。

### 助手

在当前空 Compose 项目基础上实现了“界隙书房”大厅原型：

- 深夜墨色、琥珀火光、雾白文字和紫色心智层。
- 中央“记忆避难所”叙事核心，显示“我还记得她的名字”。
- 右上方“雾中的无名门”，显示第三幕和悬念文案。
- 左侧“余响”、右侧“行囊”、底部“誓言”三个可展开互动对象。
- 余响面板展示过去选择，行囊面板展示遗物，誓言面板展示会影响剧情的承诺。
- 加入呼吸光环、漂浮尘埃、入口漂移、进入时缩放、底部面板缓动上浮、提示气泡等动效。
- 无名门支持点击和长按；底部的进入提示也会触发进入动画。
- 用 `Canvas` 做氛围渐变、弧线和房间轮廓，用 Material 3 `Surface`、`Button`、`OutlinedButton`、`Typography` 和自定义 `ColorScheme` 做交互基础。

同时补充了 `replit.md`，记录项目定位、运行方式和后续设计约定，并修复了空项目中无法解析且没有实际使用的 KSP 插件引用。
